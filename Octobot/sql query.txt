You are working with **Oracle Database** through SQL Developer. For a reliable **text-to-SQL agent**, do not give the model only a list of table and column names. Create a small metadata catalog containing:

* Table purpose and row grain
* Column meanings and data types
* Primary and foreign keys
* Approved joins
* Business synonyms
* Security and PII classifications
* Approved SQL examples

Oracle exposes technical metadata through `ALL_TAB_COLUMNS`, table and column comments through `ALL_TAB_COMMENTS` and `ALL_COL_COMMENTS`, and key relationships through `ALL_CONSTRAINTS` and `ALL_CONS_COLUMNS`. ([Oracle Docs][1])

## 1. Recommended metadata structure

Use three primary metadata tables and one optional examples table.

### Table-level metadata

```sql
CREATE TABLE PACTDATA.AI_TABLE_METADATA
(
    OWNER_NAME               VARCHAR2(128)  NOT NULL,
    TABLE_NAME               VARCHAR2(128)  NOT NULL,
    BUSINESS_LABEL           VARCHAR2(200)  NOT NULL,

    TECHNICAL_COMMENT        VARCHAR2(4000),
    BUSINESS_DESCRIPTION     VARCHAR2(4000),
    ROW_GRAIN                VARCHAR2(1000),

    PRIMARY_KEY_COLUMNS      VARCHAR2(1000),
    DEFAULT_DATE_COLUMN      VARCHAR2(128),

    ENABLED_FLAG             CHAR(1) DEFAULT 'Y' NOT NULL,
    SECURITY_CLASSIFICATION VARCHAR2(30) DEFAULT 'INTERNAL',
    AGENT_RULES              CLOB,

    LAST_REFRESHED_AT        TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,

    CONSTRAINT PK_AI_TABLE_METADATA
        PRIMARY KEY (OWNER_NAME, TABLE_NAME),

    CONSTRAINT CK_AI_TABLE_ENABLED
        CHECK (ENABLED_FLAG IN ('Y', 'N'))
);
```

### Column-level metadata

```sql
CREATE TABLE PACTDATA.AI_COLUMN_METADATA
(
    OWNER_NAME             VARCHAR2(128) NOT NULL,
    TABLE_NAME             VARCHAR2(128) NOT NULL,
    COLUMN_NAME            VARCHAR2(128) NOT NULL,

    ORDINAL_POSITION       NUMBER,
    DATA_TYPE_TEXT         VARCHAR2(500),
    NULLABLE_FLAG          CHAR(1),

    IS_PRIMARY_KEY         CHAR(1) DEFAULT 'N',
    IS_FOREIGN_KEY         CHAR(1) DEFAULT 'N',

    TECHNICAL_COMMENT      VARCHAR2(4000),
    BUSINESS_DESCRIPTION   VARCHAR2(4000),

    SEMANTIC_TYPE          VARCHAR2(50),
    SYNONYMS               VARCHAR2(1000),
    VALUE_FORMAT           VARCHAR2(500),

    -- Only store non-sensitive examples such as:
    -- OPEN, CLOSED, PENDING
    EXAMPLE_VALUES         VARCHAR2(2000),

    PII_CLASSIFICATION     VARCHAR2(30) DEFAULT 'UNCLASSIFIED',
    ALLOWED_FOR_AGENT      CHAR(1) DEFAULT 'Y' NOT NULL,

    LAST_REFRESHED_AT      TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,

    CONSTRAINT PK_AI_COLUMN_METADATA
        PRIMARY KEY (OWNER_NAME, TABLE_NAME, COLUMN_NAME),

    CONSTRAINT FK_AI_COLUMN_TABLE
        FOREIGN KEY (OWNER_NAME, TABLE_NAME)
        REFERENCES PACTDATA.AI_TABLE_METADATA
        (OWNER_NAME, TABLE_NAME)
        ON DELETE CASCADE,

    CONSTRAINT CK_AI_COLUMN_ALLOWED
        CHECK (ALLOWED_FOR_AGENT IN ('Y', 'N')),

    CONSTRAINT CK_AI_COLUMN_PK
        CHECK (IS_PRIMARY_KEY IN ('Y', 'N')),

    CONSTRAINT CK_AI_COLUMN_FK
        CHECK (IS_FOREIGN_KEY IN ('Y', 'N'))
);
```

## 2. Load your seven tables

The following script creates the table-level metadata for the tables shown in your screenshot.

Oracle normally stores unquoted object names in uppercase. Therefore, names such as `Citi_Data_NLP_Reporting` will normally appear as `CITI_DATA_NLP_REPORTING`.

```sql
MERGE INTO PACTDATA.AI_TABLE_METADATA TARGET
USING
(
    WITH TARGET_TABLES
         (OWNER_NAME, TABLE_NAME, BUSINESS_LABEL) AS
    (
        SELECT 'PACTDATA',
               'PACT_HISTORY_WORK',
               'Case History'
        FROM DUAL

        UNION ALL

        SELECT 'PACTDATA',
               'EMT_WORK_HISTORY',
               'Email History'
        FROM DUAL

        UNION ALL

        SELECT 'PACTDATA',
               'CITI_DATA_AI_INSIGHTS',
               'AI Insights'
        FROM DUAL

        UNION ALL

        SELECT 'PACTDATA',
               'CITI_DATA_NLP_REPORTING',
               'NLP Reporting'
        FROM DUAL

        UNION ALL

        SELECT 'PACTDATA',
               'CITI_DATA_NLP_CASEDATA',
               'NLP Reporting Case Data'
        FROM DUAL

        UNION ALL

        SELECT 'PACTDATA',
               'PACT_WORK',
               'Case Data'
        FROM DUAL

        UNION ALL

        SELECT 'PACTDATA',
               'CITI_DATA_LOG',
               'API Invocation Logs'
        FROM DUAL
    ),
    PK_INFORMATION AS
    (
        SELECT
            C.OWNER,
            C.TABLE_NAME,
            LISTAGG(CC.COLUMN_NAME, ', ')
                WITHIN GROUP (ORDER BY CC.POSITION)
                AS PRIMARY_KEY_COLUMNS
        FROM ALL_CONSTRAINTS C
        JOIN ALL_CONS_COLUMNS CC
          ON CC.OWNER = C.OWNER
         AND CC.CONSTRAINT_NAME = C.CONSTRAINT_NAME
         AND CC.TABLE_NAME = C.TABLE_NAME
        WHERE C.CONSTRAINT_TYPE = 'P'
        GROUP BY
            C.OWNER,
            C.TABLE_NAME
    )
    SELECT
        T.OWNER_NAME,
        T.TABLE_NAME,
        T.BUSINESS_LABEL,
        TC.COMMENTS AS TECHNICAL_COMMENT,
        PK.PRIMARY_KEY_COLUMNS
    FROM TARGET_TABLES T
    LEFT JOIN ALL_TAB_COMMENTS TC
      ON TC.OWNER = T.OWNER_NAME
     AND TC.TABLE_NAME = T.TABLE_NAME
    LEFT JOIN PK_INFORMATION PK
      ON PK.OWNER = T.OWNER_NAME
     AND PK.TABLE_NAME = T.TABLE_NAME
) SOURCE
ON
(
    TARGET.OWNER_NAME = SOURCE.OWNER_NAME
    AND TARGET.TABLE_NAME = SOURCE.TABLE_NAME
)
WHEN MATCHED THEN
    UPDATE SET
        TARGET.BUSINESS_LABEL = SOURCE.BUSINESS_LABEL,
        TARGET.TECHNICAL_COMMENT = SOURCE.TECHNICAL_COMMENT,
        TARGET.PRIMARY_KEY_COLUMNS = SOURCE.PRIMARY_KEY_COLUMNS,
        TARGET.LAST_REFRESHED_AT = SYSTIMESTAMP
WHEN NOT MATCHED THEN
    INSERT
    (
        OWNER_NAME,
        TABLE_NAME,
        BUSINESS_LABEL,
        TECHNICAL_COMMENT,
        BUSINESS_DESCRIPTION,
        PRIMARY_KEY_COLUMNS,
        ENABLED_FLAG,
        LAST_REFRESHED_AT
    )
    VALUES
    (
        SOURCE.OWNER_NAME,
        SOURCE.TABLE_NAME,
        SOURCE.BUSINESS_LABEL,
        SOURCE.TECHNICAL_COMMENT,
        SOURCE.TECHNICAL_COMMENT,
        SOURCE.PRIMARY_KEY_COLUMNS,
        'Y',
        SYSTIMESTAMP
    );

COMMIT;
```

## 3. Load column metadata automatically

```sql
MERGE INTO PACTDATA.AI_COLUMN_METADATA TARGET
USING
(
    WITH TARGET_TABLES AS
    (
        SELECT 'PACTDATA' OWNER_NAME, 'PACT_HISTORY_WORK' TABLE_NAME FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'EMT_WORK_HISTORY' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_AI_INSIGHTS' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_NLP_REPORTING' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_NLP_CASEDATA' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'PACT_WORK' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_LOG' FROM DUAL
    ),
    PK_COLUMNS AS
    (
        SELECT DISTINCT
            C.OWNER,
            C.TABLE_NAME,
            CC.COLUMN_NAME
        FROM ALL_CONSTRAINTS C
        JOIN ALL_CONS_COLUMNS CC
          ON CC.OWNER = C.OWNER
         AND CC.CONSTRAINT_NAME = C.CONSTRAINT_NAME
         AND CC.TABLE_NAME = C.TABLE_NAME
        WHERE C.CONSTRAINT_TYPE = 'P'
    ),
    FK_COLUMNS AS
    (
        SELECT DISTINCT
            C.OWNER,
            C.TABLE_NAME,
            CC.COLUMN_NAME
        FROM ALL_CONSTRAINTS C
        JOIN ALL_CONS_COLUMNS CC
          ON CC.OWNER = C.OWNER
         AND CC.CONSTRAINT_NAME = C.CONSTRAINT_NAME
         AND CC.TABLE_NAME = C.TABLE_NAME
        WHERE C.CONSTRAINT_TYPE = 'R'
    )
    SELECT
        C.OWNER AS OWNER_NAME,
        C.TABLE_NAME,
        C.COLUMN_NAME,
        C.COLUMN_ID AS ORDINAL_POSITION,

        CASE
            WHEN C.DATA_TYPE IN
                 ('VARCHAR2', 'CHAR', 'NVARCHAR2', 'NCHAR')
            THEN
                C.DATA_TYPE || '(' || C.CHAR_LENGTH ||
                CASE
                    WHEN C.CHAR_USED = 'C' THEN ' CHAR'
                    ELSE ' BYTE'
                END || ')'

            WHEN C.DATA_TYPE = 'NUMBER'
                 AND C.DATA_PRECISION IS NOT NULL
            THEN
                'NUMBER(' || C.DATA_PRECISION ||
                CASE
                    WHEN C.DATA_SCALE IS NOT NULL
                    THEN ',' || C.DATA_SCALE
                END || ')'

            WHEN C.DATA_TYPE = 'RAW'
            THEN
                'RAW(' || C.DATA_LENGTH || ')'

            ELSE C.DATA_TYPE
        END AS DATA_TYPE_TEXT,

        C.NULLABLE AS NULLABLE_FLAG,

        CASE
            WHEN PK.COLUMN_NAME IS NOT NULL THEN 'Y'
            ELSE 'N'
        END AS IS_PRIMARY_KEY,

        CASE
            WHEN FK.COLUMN_NAME IS NOT NULL THEN 'Y'
            ELSE 'N'
        END AS IS_FOREIGN_KEY,

        COMMENTS.COMMENTS AS TECHNICAL_COMMENT,

        CASE
            WHEN PK.COLUMN_NAME IS NOT NULL
              OR FK.COLUMN_NAME IS NOT NULL
            THEN 'IDENTIFIER'

            WHEN C.DATA_TYPE = 'DATE'
              OR C.DATA_TYPE LIKE 'TIMESTAMP%'
            THEN 'DATETIME'

            WHEN C.DATA_TYPE = 'NUMBER'
            THEN 'NUMBER'

            ELSE 'TEXT'
        END AS SEMANTIC_TYPE

    FROM ALL_TAB_COLUMNS C

    JOIN TARGET_TABLES T
      ON T.OWNER_NAME = C.OWNER
     AND T.TABLE_NAME = C.TABLE_NAME

    LEFT JOIN ALL_COL_COMMENTS COMMENTS
      ON COMMENTS.OWNER = C.OWNER
     AND COMMENTS.TABLE_NAME = C.TABLE_NAME
     AND COMMENTS.COLUMN_NAME = C.COLUMN_NAME

    LEFT JOIN PK_COLUMNS PK
      ON PK.OWNER = C.OWNER
     AND PK.TABLE_NAME = C.TABLE_NAME
     AND PK.COLUMN_NAME = C.COLUMN_NAME

    LEFT JOIN FK_COLUMNS FK
      ON FK.OWNER = C.OWNER
     AND FK.TABLE_NAME = C.TABLE_NAME
     AND FK.COLUMN_NAME = C.COLUMN_NAME
) SOURCE
ON
(
    TARGET.OWNER_NAME = SOURCE.OWNER_NAME
    AND TARGET.TABLE_NAME = SOURCE.TABLE_NAME
    AND TARGET.COLUMN_NAME = SOURCE.COLUMN_NAME
)
WHEN MATCHED THEN
    UPDATE SET
        TARGET.ORDINAL_POSITION = SOURCE.ORDINAL_POSITION,
        TARGET.DATA_TYPE_TEXT = SOURCE.DATA_TYPE_TEXT,
        TARGET.NULLABLE_FLAG = SOURCE.NULLABLE_FLAG,
        TARGET.IS_PRIMARY_KEY = SOURCE.IS_PRIMARY_KEY,
        TARGET.IS_FOREIGN_KEY = SOURCE.IS_FOREIGN_KEY,
        TARGET.TECHNICAL_COMMENT = SOURCE.TECHNICAL_COMMENT,
        TARGET.LAST_REFRESHED_AT = SYSTIMESTAMP
WHEN NOT MATCHED THEN
    INSERT
    (
        OWNER_NAME,
        TABLE_NAME,
        COLUMN_NAME,
        ORDINAL_POSITION,
        DATA_TYPE_TEXT,
        NULLABLE_FLAG,
        IS_PRIMARY_KEY,
        IS_FOREIGN_KEY,
        TECHNICAL_COMMENT,
        BUSINESS_DESCRIPTION,
        SEMANTIC_TYPE,
        ALLOWED_FOR_AGENT,
        LAST_REFRESHED_AT
    )
    VALUES
    (
        SOURCE.OWNER_NAME,
        SOURCE.TABLE_NAME,
        SOURCE.COLUMN_NAME,
        SOURCE.ORDINAL_POSITION,
        SOURCE.DATA_TYPE_TEXT,
        SOURCE.NULLABLE_FLAG,
        SOURCE.IS_PRIMARY_KEY,
        SOURCE.IS_FOREIGN_KEY,
        SOURCE.TECHNICAL_COMMENT,
        SOURCE.TECHNICAL_COMMENT,
        SOURCE.SEMANTIC_TYPE,
        'Y',
        SYSTIMESTAMP
    );

COMMIT;
```

## 4. Extract defined foreign-key relationships

```sql
MERGE INTO PACTDATA.AI_RELATIONSHIP_METADATA TARGET
USING
(
    WITH TARGET_TABLES AS
    (
        SELECT 'PACTDATA' OWNER_NAME, 'PACT_HISTORY_WORK' TABLE_NAME FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'EMT_WORK_HISTORY' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_AI_INSIGHTS' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_NLP_REPORTING' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_NLP_CASEDATA' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'PACT_WORK' FROM DUAL
        UNION ALL
        SELECT 'PACTDATA', 'CITI_DATA_LOG' FROM DUAL
    )
    SELECT
        FK.OWNER AS SOURCE_OWNER,
        FK.TABLE_NAME AS SOURCE_TABLE,
        FK.CONSTRAINT_NAME AS RELATIONSHIP_NAME,

        LISTAGG(FKC.COLUMN_NAME, ', ')
            WITHIN GROUP (ORDER BY FKC.POSITION)
            AS SOURCE_COLUMNS,

        PK.OWNER AS TARGET_OWNER,
        PK.TABLE_NAME AS TARGET_TABLE,

        LISTAGG(PKC.COLUMN_NAME, ', ')
            WITHIN GROUP (ORDER BY PKC.POSITION)
            AS TARGET_COLUMNS,

        LISTAGG(
            'SRC.' || FKC.COLUMN_NAME ||
            ' = TGT.' || PKC.COLUMN_NAME,
            ' AND '
        ) WITHIN GROUP (ORDER BY FKC.POSITION)
          AS JOIN_EXPRESSION

    FROM ALL_CONSTRAINTS FK

    JOIN TARGET_TABLES T
      ON T.OWNER_NAME = FK.OWNER
     AND T.TABLE_NAME = FK.TABLE_NAME

    JOIN ALL_CONS_COLUMNS FKC
      ON FKC.OWNER = FK.OWNER
     AND FKC.CONSTRAINT_NAME = FK.CONSTRAINT_NAME
     AND FKC.TABLE_NAME = FK.TABLE_NAME

    JOIN ALL_CONSTRAINTS PK
      ON PK.OWNER = FK.R_OWNER
     AND PK.CONSTRAINT_NAME = FK.R_CONSTRAINT_NAME

    JOIN ALL_CONS_COLUMNS PKC
      ON PKC.OWNER = PK.OWNER
     AND PKC.CONSTRAINT_NAME = PK.CONSTRAINT_NAME
     AND PKC.TABLE_NAME = PK.TABLE_NAME
     AND PKC.POSITION = FKC.POSITION

    WHERE FK.CONSTRAINT_TYPE = 'R'
      AND FK.STATUS = 'ENABLED'

    GROUP BY
        FK.OWNER,
        FK.TABLE_NAME,
        FK.CONSTRAINT_NAME,
        PK.OWNER,
        PK.TABLE_NAME
) SOURCE
ON
(
    TARGET.SOURCE_OWNER = SOURCE.SOURCE_OWNER
    AND TARGET.SOURCE_TABLE = SOURCE.SOURCE_TABLE
    AND TARGET.RELATIONSHIP_NAME = SOURCE.RELATIONSHIP_NAME
)
WHEN MATCHED THEN
    UPDATE SET
        TARGET.SOURCE_COLUMNS = SOURCE.SOURCE_COLUMNS,
        TARGET.TARGET_OWNER = SOURCE.TARGET_OWNER,
        TARGET.TARGET_TABLE = SOURCE.TARGET_TABLE,
        TARGET.TARGET_COLUMNS = SOURCE.TARGET_COLUMNS,
        TARGET.JOIN_EXPRESSION = SOURCE.JOIN_EXPRESSION,
        TARGET.CARDINALITY = 'MANY_TO_ONE',
        TARGET.VERIFIED_FLAG = 'Y',
        TARGET.LAST_REFRESHED_AT = SYSTIMESTAMP
WHEN NOT MATCHED THEN
    INSERT
    (
        SOURCE_OWNER,
        SOURCE_TABLE,
        RELATIONSHIP_NAME,
        SOURCE_COLUMNS,
        TARGET_OWNER,
        TARGET_TABLE,
        TARGET_COLUMNS,
        JOIN_EXPRESSION,
        CARDINALITY,
        VERIFIED_FLAG,
        LAST_REFRESHED_AT
    )
    VALUES
    (
        SOURCE.SOURCE_OWNER,
        SOURCE.SOURCE_TABLE,
        SOURCE.RELATIONSHIP_NAME,
        SOURCE.SOURCE_COLUMNS,
        SOURCE.TARGET_OWNER,
        SOURCE.TARGET_TABLE,
        SOURCE.TARGET_COLUMNS,
        SOURCE.JOIN_EXPRESSION,
        'MANY_TO_ONE',
        'Y',
        SYSTIMESTAMP
    );

COMMIT;
```

## 5. Add relationships that do not have database constraints

Many reporting or staging tables do not have physical foreign keys. In that situation, add the joins manually after confirming them with the data owner.

Example template:

```sql

(

    'PACTDATA',

    '<CASE_KEY_COLUMN>',
```

Replace `<CASE_KEY_COLUMN>` with the actual common case identifier.

## 6. Add business descriptions

Technical metadata alone is usually insufficient. For example, a column named `STATUS` does not tell the agent whether it means case status, email status, API status, or processing status.

Oracle's `COMMENT ON TABLE` and `COMMENT ON COLUMN` statements store documentation in the database dictionary, where it can be read through the comments views. ([Oracle Docs][2])

Example:

```sql
COMMENT ON TABLE PACTDATA.PACT_WORK IS
'Current case-level data. Expected grain is one row per case.';

COMMENT ON COLUMN PACTDATA.PACT_WORK.CASE_ID IS
'Unique identifier of a case used to join with case history and NLP case data.';
```

You can also update the agent metadata directly:

```sql
UPDATE PACTDATA.AI_TABLE_METADATA
SET
    BUSINESS_DESCRIPTION =
        'Contains current case information used for case-level reporting.',
    ROW_GRAIN =
        'One row per case',
    DEFAULT_DATE_COLUMN =
        '<ACTUAL_DATE_COLUMN>',
    AGENT_RULES =
        'Use this table for current case attributes. Use the history table for case status changes.'
WHERE OWNER_NAME = 'PACTDATA'
  AND TABLE_NAME = 'PACT_WORK';
```

Column enrichment example:

```sql
UPDATE PACTDATA.AI_COLUMN_METADATA
SET
    BUSINESS_DESCRIPTION =
        'Current business status of the case.',
    SYNONYMS =
        'case status,current status,state',
    SEMANTIC_TYPE =
        'STATUS',
    EXAMPLE_VALUES =
        'OPEN,CLOSED,PENDING'
WHERE OWNER_NAME = 'PACTDATA'
  AND TABLE_NAME = 'PACT_WORK'
  AND COLUMN_NAME = '<ACTUAL_STATUS_COLUMN>';
```

Do not store real names, email addresses, case descriptions, tokens, or other sensitive values in `EXAMPLE_VALUES`.

## 7. Create a flat view for the agent

```sql
CREATE OR REPLACE VIEW PACTDATA.AI_AGENT_SCHEMA_CONTEXT_V AS
SELECT
    T.OWNER_NAME,
    T.TABLE_NAME,
    T.BUSINESS_LABEL,
    NVL(T.BUSINESS_DESCRIPTION, T.TECHNICAL_COMMENT)
        AS TABLE_DESCRIPTION,
    T.ROW_GRAIN,
    T.PRIMARY_KEY_COLUMNS,
    T.DEFAULT_DATE_COLUMN,
    T.AGENT_RULES,

    C.COLUMN_NAME,
    C.ORDINAL_POSITION,
    C.DATA_TYPE_TEXT,
    C.NULLABLE_FLAG,
    C.IS_PRIMARY_KEY,
    C.IS_FOREIGN_KEY,
    NVL(C.BUSINESS_DESCRIPTION, C.TECHNICAL_COMMENT)
        AS COLUMN_DESCRIPTION,
    C.SEMANTIC_TYPE,
    C.SYNONYMS,
    C.VALUE_FORMAT,
    C.EXAMPLE_VALUES,
    C.PII_CLASSIFICATION

FROM PACTDATA.AI_TABLE_METADATA T
JOIN PACTDATA.AI_COLUMN_METADATA C
  ON C.OWNER_NAME = T.OWNER_NAME
 AND C.TABLE_NAME = T.TABLE_NAME

WHERE T.ENABLED_FLAG = 'Y'
  AND C.ALLOWED_FOR_AGENT = 'Y';
```

The agent application can query this view instead of querying the complete Oracle data dictionary.

## 8. How the text-to-SQL agent should work

A reliable flow is:

```text
User question
      ↓
Find likely tables from AI_TABLE_METADATA
      ↓
Retrieve columns for those tables
      ↓
Retrieve verified relationships
      ↓
LLM generates Oracle SELECT statement
      ↓
SQL validator checks statement
      ↓
Execute using read-only database account
      ↓
Return query results to LLM
      ↓
LLM explains the results to the user
```

Do not send the metadata for every database table in every prompt. First select two or three likely tables from their labels and descriptions, and then retrieve their detailed column metadata.

## 9. Suggested SQL-generation prompt

```text
You are an Oracle SQL query generator.

Generate SQL only from the supplied table, column, and relationship metadata.

Rules:
1. Never invent a table or column.
2. Generate exactly one SELECT statement or WITH...SELECT statement.
3. Do not generate INSERT, UPDATE, DELETE, MERGE, ALTER, DROP,
   TRUNCATE, GRANT, EXECUTE, or PL/SQL.
4. Use only relationships marked VERIFIED_FLAG = 'Y'.
5. Use explicit JOIN syntax.
6. Use bind variables for user-provided values.
7. Do not select columns marked as disallowed or sensitive.
8. For non-aggregate detail queries, return no more than 200 rows.
9. When the request is ambiguous, return a clarification question
   rather than guessing.
10. Return:
    - generated_sql
    - bind_parameters
    - tables_used
    - short_explanation
```

## 10. Important execution controls

The SQL validator should independently enforce:

* Statement starts with `SELECT` or `WITH`
* Only enabled tables are referenced
* No multiple statements or semicolons in the middle
* No DDL or DML keywords
* No access to system schemas
* Bind parameters are used for user values
* Detail queries use `FETCH FIRST 200 ROWS ONLY`
* Execution uses a read-only Oracle account
* Query timeout is configured
* Sensitive columns have `ALLOWED_FOR_AGENT = 'N'`

`DBMS_METADATA.GET_DDL` can also retrieve an object's complete creation DDL, but full DDL is better used as an administrative reference than as the only semantic context for the agent. ([Oracle Docs][3])
